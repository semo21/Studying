# 236. Lowest Common Ancestor of a Binary Tree

## Difficulty
- Medium

## Point
- Use recursion to search both left and right subtrees.
- Return `root` if the current node is `p` or `q`.
- If both left and right return non-null, the current node is the LCA.
- If only one side returns non-null, pass that result upward.

## Code
```c++
/**
 * Definition for a binary tree node.
 * struct TreeNode {
 *     int val;
 *     TreeNode *left;
 *     TreeNode *right;
 *     TreeNode(int x) : val(x), left(NULL), right(NULL) {}
 * };
 */
class Solution {
public:
    TreeNode* lowestCommonAncestor(TreeNode* root, TreeNode* p, TreeNode* q) {
        if(root == nullptr || root == p || root == q) return root;

        TreeNode* left = lowestCommonAncestor(root->left, p, q);
        TreeNode* right = lowestCommonAncestor(root->right, p, q);

        if(left == nullptr){
            return right;
        }else if(right == nullptr){
            return left;
        }else{
            return root;
        }
    }
};
```

### Constraints
- The number of nodes in the tree is in the range [2, 105].
- -109 <= Node.val <= 109
- All Node.val are unique.
- p != q
- p and q will exist in the tree.

## Explanation
- If `root` is `nullptr`, return `nullptr`.
- If `root` is either `p` or `q`, return `root`.
- Recursively search the left and right subtrees.
- If both sides return non-null, it means `p` and `q` were found on different sides, so the current node is the LCA.
- If only one side returns non-null, return that node upward.
- The return value can mean `nullptr`, one of the target nodes, or the already found LCA.

## Complexity
### Time Complexity: O(n)
- Each node is visited at most once.

### Space Complexity: O(n)
- Recursive calls use the call stack.
- In the worst case, the tree can be skewed, so the recursion depth can be n.
