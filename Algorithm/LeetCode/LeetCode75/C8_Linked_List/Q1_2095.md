# 2095. Delete the Middle Node of a Linked List

## Difficulty
- Medium

## Point
- Linked List does not support random access.
- To reach a specific node, nodes must be traversed one by one.
- Remove a node by changing the previous node's `next` pointer.

## Code
```c++
/**
 * Definition for singly-linked list.
 * struct ListNode {
 *     int val;
 *     ListNode *next;
 *     ListNode() : val(0), next(nullptr) {}
 *     ListNode(int x) : val(x), next(nullptr) {}
 *     ListNode(int x, ListNode *next) : val(x), next(next) {}
 * };
 */
class Solution {
public:
    ListNode* deleteMiddle(ListNode* head) {
        if(head->next == nullptr){
            return nullptr;
        }

        int len = 0;
        ListNode* node = head;
        
        while(node != nullptr){
            len++;
            node = node->next;
        }        
        
        int n = len / 2;

        node = head;        
        for(int i = 0; i < n-1; ++i){
            node = node->next;
        }
        node->next = node->next->next;
        
        return head;
    }
};
```

### Constraints
- The number of nodes in the list is in the range [1, 10^5].
- 1 <= Node.val <= 10^5

## Explanation
- If the list contains only one node, return `nullptr`.
- Traverse the linked list once to calculate its length.
- Find the index of the middle node using `len / 2`.
- Traverse the list again until reaching the node before the middle node.
- Remove the middle node by updating the `next` pointer.
- Return the original head node.

## Complexity
### Time Complexity: O(n)
- The linked list is traversed once to calculate its length.
- The list is traversed again to reach the node before the middle node.
- Therefore, the total time complexity is O(n).

### Space Complexity: O(1)
- Only a few variables are used regardless of the input size.
