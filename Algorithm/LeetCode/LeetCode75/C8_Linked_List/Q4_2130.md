# 2130. Maximum Twin Sum of a Linked List

## Difficulty
- Medium

## Point
- Use `slow` and `fast` pointers to find the start of the second half.
- Reverse the second half of the linked list from the `slow` node.
- Compare nodes from the first half and the reversed second half to find the maximum twin sum.

## Code
```c++
/**
 * Definition for singly-linked list.
 * struct ListNode {
 *     int val;
 *     ListNode *next;
 *     ListNode() : val(0), next(nullptr) {}
 *     ListNode(int x) : val(x), next(nullptr) {}
 *     ListNode(int x, ListNode *next) : val(x), next(next) {}
 * };
 */
class Solution {
public:
    int pairSum(ListNode* head) {
        ListNode* slow = head;
        ListNode* fast = head;

        while(fast != nullptr && fast->next != nullptr){
            slow = slow->next;
            fast = fast->next->next;
        }

        ListNode* prev = nullptr;
        ListNode* cur = slow;
        while(cur != nullptr){
            ListNode* nextNode = cur->next;
            cur->next = prev;
            prev = cur;
            cur = nextNode;            
        }

        ListNode* left = head;
        ListNode* right = prev;

        int answer = 0;

        while(right != nullptr){
            answer = max(answer, left->val + right->val);
            left = left->next;
            right = right->next;
        }
        
        return answer;
    }
};
```

### Constraints
- The number of nodes in the list is an even integer in the range [2, 105].
- 1 <= Node.val <= 105

## Explanation
- Use `slow` and `fast` pointers to find the middle of the linked list.
- Since the list length is even, `slow` points to the first node of the second half.
- Reverse the second half of the linked list in-place.
- Use `left` from `head` and `right` from the reversed second half.
- Traverse both halves together and update the maximum twin sum.

## Complexity
### Time Complexity: O(n)
- Use `slow` and `fast` pointers to find the middle of the linked list.
- Reverse the second half of the linked list.
- Traverse both halves to calculate twin sums.
- These steps are all linear, so the total time complexity is O(n).

### Space Complexity: O(1)
- Only a fixed number of pointer variables are used.
- The second half is reversed in-place without using an additional array.
