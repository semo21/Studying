# 206. Reverse Linked List

## Difficulty
- Easy

## Point
- Connect current node's next pointer to previous node.
- Disconnect existing pinter connection.

## Code
```c++
/**
 * Definition for singly-linked list.
 * struct ListNode {
 *     int val;
 *     ListNode *next;
 *     ListNode() : val(0), next(nullptr) {}
 *     ListNode(int x) : val(x), next(nullptr) {}
 *     ListNode(int x, ListNode *next) : val(x), next(next) {}
 * };
 */

 // Iterative
class Solution {
public:
    ListNode* reverseList(ListNode* head) {        
        ListNode* prev = nullptr;
        ListNode* cur = head;
        while(cur != nullptr){         
            ListNode* next = cur->next;   
            
            cur->next = prev;
            prev = cur;
            cur = next;            
        }

        return prev;
    }
};

// Recursive
class Solution {
public:
    ListNode* reverseList(ListNode* head) {        
        if(head == nullptr || head->next == nullptr){
            return head;
        }

        ListNode* newHead = reverseList(head->next);
        head->next->next = head;
        head->next = nullptr;

        return newHead;
    }    
};
```

### Constraints
- The number of nodes in the list is the range [0, 5000].
- -5000 <= Node.val <= 5000

## Explanation
### Iterative
- Use three pointer: `prev`, `cur`, and `next`.
- Save the next node before modifying the link.
- Reverse the current node's pointer by connecting it to the previous node.
- Mode all pointers one step forward.
- After the traversal, `prev` becomes the new head of the reversed list.

### Recursive
- Recursively reverse the rest of the list starting from `head->next`.
- Connect the next node back to the current node using `head->next->next = head`.
- Disconnect the original link using `head->next = nullptr.`
- Return the new head obtained from the recursive call.

## Complexity
### Time Complexity
#### Iterative: O(n)
- Traverse the linked list once.

#### Recursive: O(n)
- Only a few pointer variables are used.

### Space Complexity
#### Iterative: O(1)
- Each node is visited once.

#### Recursive: O(n)
- Recursive calls are stored in the call stack.
