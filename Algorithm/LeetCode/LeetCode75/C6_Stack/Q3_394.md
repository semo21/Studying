# 394. Decode String

## Difficulty
- Medium

## Point
- Use stacks to store previous strings and repeat counts.
- When `[` is found, save the current state.
- When `]` is found, decode the current substring and restore the previous state.

## Code
```c++
class Solution {
public:
    string decodeString(string s) {
        string current;
        int num = 0;
        
        stack<string> strSt;
        stack<int> numSt;

        for(char c : s){
            if(isdigit(c)){
                num = 10 * num + (c - '0');
            }else if(c == '['){
                numSt.push(num);
                strSt.push(current);

                num = 0;
                current = "";
            }else if(c == ']'){
                int repeat = numSt.top();
                numSt.pop();

                string prev = strSt.top();
                strSt.pop();

                string temp = current;
                current = prev;

                for(int i = 0 ; i < repeat; ++i){
                    current += temp;
                }
            }else{
                current += c;
            }
            
        }

        return current;        
    }
};
```

### Constraints
- 1 <= s.length <= 30
- s consists of lowercase English letters, digits, and square brackets '[]'.
- s is guaranteed to be a valid input.
- All the integers in s are in the range [1, 300].

## Explanation
- Traverse the string character by character.
- If the current character is a digit, build the repeat number in `num`.
- If `[` is found, push the current repeat number and current string into stacks.
- Then reset `num` and `current` to start decoding the inner substring.
- If `]` is found, pop the previous string and repeat count from the stacks.
- Repeat the current decoded substring and append it to previous string.
- This stack-based approach handles nested brackets correctly.

## Complexity
### Time Complexity: O(n)
- The input string is traversed once.
- Each character is processed using stack operations or string concatenation.

Note:
> If the decoded output length is considered, the time complexity can be O(m), where `m` is the length of the decoded string.

### Space Complextity: O(n)
- The stack store previous strings and repeat counts for nested brackets.
- The current string also stores decoded characters.
