# 735. Asteroid Collision

## Difficulty
- Medium

## Point
- Use a vector as a stack.
- A collision happens only when the previous asteroid moves right and the current asteroid moves left.

## Code
```c++
class Solution {
public:
    vector<int> asteroidCollision(vector<int>& asteroids) {
        vector<int> result;

        for(int a : asteroids){
            bool destroyed = false;

            while(!result.empty() && result.back() > 0 && a < 0){
                if(result.back() < -a){
                    result.pop_back();
                }else if(result.back() == -a){
                    result.pop_back();
                    destroyed = true;
                    break;
                }else{
                    destroyed = true;
                    break;
                }
            }

            if(!destroyed){
                result.push_back(a);
            }
        }

        return result;
    }
};
```

### Constraints
- 2 <= asteroids.length <= 104
- -1000 <= asteroids[i] <= 1000
- asteroids[i] != 0

## Explanation
- Store surviving asteroids in the `result` vector.
- For each asteroid `a`, compare it with `result.back()` only when `result.back() > 0` and `a < 0`.
- If `abs(a)` is greater than `result.back()`, remove `result.back()` and continue checking.
- If both sizes are equal, remove `result.back()` and mark the current asteroid as destroyed.
- If `result.back()` is greater, only the current asteroid is destroyed.
- If the current asteroid is not destroyed, append it to `result`.

## Complexity
### Time Complexity: O(n^2)
- Each asteroid is pushed into `result` at most once.
- Each asteroid is popped from `result` at most once.
- Therefore, the total number of operations is linear.

### Space Complextity: O(n)
- In the worst case, all asteroids survive and are stored in `result`.
