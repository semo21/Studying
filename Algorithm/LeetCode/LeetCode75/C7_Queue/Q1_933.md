# 933. Number of Recent Calls

## Difficulty
- Easy

## Point
- Each test case will call ping with strictly increasing values of t.

## Code
```c++
class RecentCounter {
public:
    queue<int> que;
    RecentCounter() {
        
    }
    
    int ping(int t) {
        que.push(t);

        while(!que.empty() && que.front() < t - 3000){
            que.pop();
        }

        return que.size();
    }
};

/**
 * Your RecentCounter object will be instantiated and called as such:
 * RecentCounter* obj = new RecentCounter();
 * int param_1 = obj->ping(t);
 */
```

### Constraints
- 1 <= t <= 109
- Each test case will call ping with strictly increasing values of t.
- At most 104 calls will be made to ping.

## Explanation
- Since `t` is strictly increasing, older requests are always located at the front of the queue.
- Add the current request time `t` to the queue.
- Remove all requests smaller than `t - 3000`.
- The remaining queue size is the number of recent requests.

## Complexity
### Time Complexity: O(n) /  Amortized: O(1)
- Each request is pushed into the queue once.
- Each request is popped from the queue at most once.
- Therefore, across all calls, the total work is linear.

### Space Complextity: O(n)
- The queue may store up to `n` request times.
