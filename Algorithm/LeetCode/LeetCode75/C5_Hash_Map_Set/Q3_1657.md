# 1657. Determine if Two Strings Are Close

## Difficulty
- Medium

## Point
- Two strings can be transformed into each other if they contain the same set of characters and the same frequency distribution.
- Since the problem only uses lowercases English letters, a fixed-size frequency array of length 26 is more efficient than a hash map.

## Code
```c++
// solution # 1
class Solution {
public:
    bool closeStrings(string word1, string word2) {
        unordered_map<char, int> freq1;
        unordered_map<char, int> freq2;

        for(char c : word1){
            freq1[c]++; 
        }

        for(char c : word2){
            freq2[c]++;
        }

        vector<int> counts1;
        vector<int> counts2;

        for(auto& p : freq1){            
            if(freq2.count(p.first) == 0)   return false;
            
            counts1.push_back(p.second);
        }
        for(auto& p : freq2){
            if(freq1.count(p.first) == 0)   return false;

            counts2.push_back(p.second);
        } 

        sort(counts1.begin(), counts1.end());
        sort(counts2.begin(), counts2.end());

        return counts1 == counts2;
    }
};

// solution # 2
class Solution {
public:
    bool closeStrings(string word1, string word2) {
        vector<int> freq1(26, 0);
        vector<int> freq2(26, 0);

        for(char c : word1){
            freq1[c - 'a']++; 
        }

        for(char c : word2){
            freq2[c - 'a']++;
        }

        for(int i = 0 ; i < 26 ; ++i){
            if((freq1[i] == 0) != (freq2[i] == 0)) {
                return false;
            }
        }

        sort(freq1.begin(), freq1.end());
        sort(freq2.begin(), freq2.end());

        return freq1 == freq2;
    }
};
```

### Constraints:
- 1 <= word1.length, word2.length <= 105
- word1 and word2 contain only lowercase English letters.
 
## Explanation
- Count the frequency of each character in both strings.
- The tow strings must contain the same set of characters. If a character exists in one string but not in the other, it is impossible to transform one string into the other.
- Compare the frequency distributions. Since characters can swap their frequencies through the allowed operations, only the frequency counts matter, not which character owns each count.
- After sorting the frequency arrays, the two strings are considered close if the sorted frequency arrays are identical.

## Complexity
### Time Complexity: O(n)
- Both strings are traversed once to count character frequencies.
- All other operations are performed on fixed-size arrays of length 26.
- Therefore, the overall time complexity is O(n).
### Space Complextity: O(1)
- Two frequency arrays of size 26 are used.
- Since the alphabet size is fixed, the extra space does not grow with the input size.
- Therefore, the space complexity is O(1).
