# 2352. Equal Row and Column Pairs

## Difficulty
- Medium

## Point
- Use each row in the grid as a key in a map
- Build each column as a vector and check how many matching rows exist.

## Code
```c++
class Solution {
public:
    int equalPairs(vector<vector<int>>& grid) {
        int n = grid.size();

        map<vector<int>, int> rows;
        for(int i = 0; i < n; ++i){
            rows[grid[i]]++;
        }

        int count = 0;

        for(int j = 0; j < n; ++j){
            vector<int> col;
            for(int row = 0; row < n; ++row){
                col.push_back(grid[row][j]);
            }

            count += rows[col];
        }

        return count;
    }
};
```

### Constraints:
- n == grid.length == grid[i].length
- 1 <= n <= 200
- 1 <= grid[i][j] <= 105
 
## Explanation
- Store each row of the grid in a map
- The key is the row vector, and the value is the number of times that row appears
- Then, build each column as a vector
- If the column vector exists in the map, add its stored count to the answer
- This works because each matching row-column pair contributes one valid pair

## Complexity
### Time Complexity: O(n^2 log n)
- These are n rows, and each row has length n
- Each row is inserted into the map
- Each column is also built and searched in the map
- Since map uses ordered keys, insertion and lookup take logarithmic time
### Space Complextity: O(n^2)
- The map can stored up to n row vectors
- Each row vector has length n
- Therefore, the extra space used by the map is O(n^2)
